========================================================================
  ANALISIS DE SEGURIDAD -- DETALLE
========================================================================

  [OK]   SQL Injection
         Todas las queries usan parametros preparados (mysql2 pool.execute
         con placeholders '?'). Ningun endpoint concatena strings directamente
         en SQL. Payloads como "' OR '1'='1" o "'; DROP TABLE" se almacenan
         como texto plano sin ejecutarse como SQL.

  [OK]   XSS (Cross-Site Scripting)
         La API devuelve exclusivamente JSON. React escapa automaticamente
         todo el contenido al renderizarlo (sin innerHTML ni
         dangerouslySetInnerHTML con datos de la API). Un payload
         "<script>alert(1)</script>" se almacena como texto plano y React
         lo muestra como cadena, no lo ejecuta.

  [OK]   Path Traversal
         Express.static sirve unicamente desde /public/uploads/.
         Cualquier intento de salir del directorio (/../../etc/passwd,
         /%2e%2e/) devuelve 404. El servidor no expone package.json, .env
         ni node_modules.

  [OK]   Autenticacion Admin
         9 vectores de bypass intentados (sin token, token vacio, token
         incorrecto, sin prefijo Bearer, header alternativo, password
         null/vacio/incorrecto, SQLi en password). Todos rechazados con 401.
         El ADMIN_KEY se compara contra el valor exacto de la variable de
         entorno, no contra la base de datos.

  [OK]   CORS
         Configurado con lista blanca ALLOWED_ORIGINS. No se acepta
         Access-Control-Allow-Origin: * en produccion. Solo los origenes
         autorizados pueden hacer peticiones con credenciales.

  [OK]   Subida de archivos (Multer)
         Valida MIME type (solo imagenes: image/*). Nombres de archivo
         generados con crypto.randomBytes(8) + extension original.
         No permite path injection ni ejecucion de archivos subidos.

  [OK]   Mass Assignment
         Campos extra en el body JSON (admin:true, role:superadmin,
         __proto__) son ignorados por Zod (validacion estricta de esquema)
         y por las queries SQL (solo se usan los campos mapeados
         explicitamente). No hay riesgo de escalacion de privilegios.

  [WARN] Rate Limiting -- NO IMPLEMENTADO
         No existe limite de peticiones por IP. Un atacante podria hacer
         fuerza bruta contra /admin/auth sin restriccion.
         RECOMENDACION: Agregar express-rate-limit antes de produccion:
           npm install express-rate-limit
           const rateLimit = require('express-rate-limit')
           app.use('/api/v1/admin/auth', rateLimit({ windowMs: 15*60*1000, max: 10 }))

  [WARN] HTTPS -- Solo HTTP en desarrollo
         En produccion es obligatorio HTTPS (Nginx + Let's Encrypt).
         Sin HTTPS el ADMIN_KEY viaja en texto plano por la red.

  [WARN] Autenticacion Admin -- Sin expiracion de sesion
         La ADMIN_KEY no expira. Si se compromete, el acceso es permanente
         hasta que se cambie manualmente en .env.
         RECOMENDACION para produccion: Migrar a JWT con expiracion de 8h.

  [WARN] Google Sheets como dependencia critica
         Si la API de Google Sheets falla, el endpoint POST /cotizacion
         devuelve 502 y no guarda nada. El usuario no puede cotizar.
         RECOMENDACION: Implementar cola de reintentos o modo fallback
         que guarde solo en MariaDB si Sheets no responde.

  [INFO] Datos sensibles en respuestas
         Los endpoints publicos no exponen campos internos (id de BD,
         sheets_fila, docx_path). Los endpoints admin si los exponen,
         pero estan protegidos por ADMIN_KEY.
